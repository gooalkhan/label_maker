# 한글 표시사항 이미지 변환 LLM 프롬프트 지침

LLM(예: ChatGPT, Claude 등)에 한글 표시사항 이미지를 업로드하고, 이 프로젝트(`label_maker`)에서 바로 렌더링 가능한 JSON 파일을 생성하도록 요청할 때 사용할 수 있는 프롬프트 내용입니다. 

아래의 내용을 복사하여 LLM에게 이미지와 함께 전달하세요.

---

## LLM 프롬프트 (이 아래부터 복사)

당신은 한국 식품의 "한글 표시사항" 라벨 이미지를 분석하고, 이를 구조화된 JSON 데이터로 변환하는 전문가입니다. 
제가 첨부한 식품 라벨 이미지를 분석하여, 아래에서 설명하는 규칙과 JSON 스키마에 맞춰 정확한 JSON 데이터를 생성해 주세요.

### 1. 사이즈 및 규격 추정 규칙
제공된 이미지에는 실제 물리적 사이즈(mm)가 명시되어 있지 않습니다. 따라서 다음 규칙에 따라 라벨의 `width`와 `height` 및 기타 속성들을 추정해야 합니다.
- **가장 작은 글씨 크기 기준:** 이미지 내에 있는 가장 작은 글자의 크기를 법적 최소 기준인 **10pt (약 3.53mm)** 로 가정합니다.
- 이 10pt 글자의 크기를 기준으로, 이미지 전체의 비율을 역산하여 `main_table`과 `nutrition_facts`(영양정보, 있는 경우)의 전체 너비(`width`, mm 단위)와 높이(`height`, mm 단위)를 합리적으로 추정하세요.
- **장평 (horizontal_scale):** 글자의 세로 길이를 1로 보았을 때의 가로 너비 비율로 확인합니다. 비율이 0.9 이하로 좁게 보일 때도, 강제로 **0.9**를 적용하세요.
- **셀 패딩 (cell_padding):** 표 안의 글자와 내부 테두리 사이의 여백 공간을 앞서 가정한 10pt(약 3.53mm) 높이와 비교하여 대략적인 mm 단위로 추정하세요. (예: 글자 높이의 1/3 정도라면 약 1.0~1.2 등)

### 2. JSON 스키마 구조
출력하는 JSON은 반드시 아래의 구조를 따라야 합니다.

```json
{
  "label_config": {
    "nutrition_mode": "bottom", // 영양정보 위치 ("bottom", "left", "right", "none" 중 택 1)
    "global_padding": 2, // 기본 패딩 (mm)
    "cell_padding": 1, // 셀 패딩 (mm)
    "border_thickness": 0.5, // 기본 테두리 두께 (mm)
    "horizontal_scale": 1.0, // 글자 장평 (기본 1.0)
    "font_family": "NanumGothic"
  },
  "main_table": {
    "width": 130, // 추정한 전체 너비 (mm)
    "height": 110, // 추정한 전체 높이 (mm)
    "cells": [
      // ... 분석한 정보 셀 객체들 ...
    ]
  },
  "nutrition_facts": {
    "width": 100, // 영양정보 너비 (없으면 생략 가능)
    "height": 100, // 영양정보 높이
    "cells": [
      // ... 영양정보 셀 객체들 ...
    ]
  }
}
```

### 3. 셀(Cell) 데이터 분석 및 작성 지침
이미지에 있는 각 정보 단락이나 표의 칸을 `cells` 배열 내의 하나의 객체로 변환하세요.

**기본 속성:**
- `header`: 항목의 제목 (예: "제품명", "원재료명"). 
  - **중요:** 만약 제목 배경이 검은색이고 글자가 흰색으로 반전되어 있다면, 반드시 `[[제품명]]`과 같이 대괄호 두 개로 감싸주세요.
- `content`: 항목의 내용.
- `no_break`: 제목(`header`)과 내용(`content`)이 중간에 줄바꿈 없이 반드시 한 줄에 이어져야 한다면 `true`로 설정하세요.
- `use_whole_line`: 해당 항목이 라벨 내에서 한 줄 전체 너비를 단독으로 사용해야 한다면 `true`로 설정하세요.
- `align`: 텍스트 정렬이 필요하다면 `"left"`, `"center"`, `"right"` 중 하나를 지정하세요 (기본값은 left).
- `indent`: 들여쓰기가 되어있다면 추정한 들여쓰기 수치(mm)를 숫자로 입력하세요.

**플레이스홀더 (마크 / 인증 로고 / 흑백 반전 박스):**
- 라벨 좌측이나 우측 등에 분리배출 마크나 영양정보 타이틀 같은 사각형 박스가 위치한다면, 다음과 같이 플레이스홀더 셀을 추가하세요:
  `{ "type": "placeholder", "label": "[[비닐류 OTHER]]", "width": 20, "height": 20, "position": "left" }`
- `position`: 플레이스홀더의 좌우 위치(`"left"` 또는 `"right"`)를 지정할 수 있습니다. (생략 시 기본값 `"right"`)
- `label`: 박스 내부의 텍스트입니다. 박스 전체 배경이 검은색이고 글자가 흰색으로 반전되어 있다면 텍스트를 `[[ ]]`로 감싸주세요 (예: `"[[영양성분]]"`). 텍스트는 박스의 가로 너비(`width`)에 맞춰 글씨 크기가 자동으로 조절되어 꽉 차게 렌더링됩니다.

### 4. 최종 출력 제한사항
- 추정 과정이나 분석 결과에 대한 설명은 전혀 필요하지 않습니다.
- 오직 렌더러가 바로 읽어들일 수 있는 **순수한 JSON 코드 텍스트**만 출력해 주세요 (마크다운 코드 블록 문법은 사용해도 됩니다).
